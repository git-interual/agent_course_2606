# Agentic AI 과정 Day 3 - 1교시
## OpenClaw + Discord 연동 실습

**일시:** 2026-06-18  
**환경:** Azure Ubuntu Pro 24.04 LTS VM  
**모델:** Microsoft Foundry - dev-gpt-5.4-mini

---

# 목표

강의에서는 Telegram 연동을 진행하였으나, 실습에서는 Discord 연동을 시도하였다.

목표 구조:

```
Discord
   ↓
OpenClaw
   ↓
Microsoft Foundry
   ↓
GPT Model
```

---

# 1. OpenClaw 설치 상태 확인

상태 확인:

```bash
openclaw status
openclaw status --deep
```

확인 결과:

```text
Gateway service : running
Discord         : ON
Discord         : OK
accounts        : 1/1
```

초기에는 다음 상태였다.

```text
discord ON SETUP
```

이 상태에서는 Discord 설정은 존재하지만 실제 연결은 완료되지 않은 상태이다.

이후:

```text
Discord ON OK
token config
accounts 1/1
```

상태가 되어 Discord 인증 자체는 성공한 것으로 확인되었다.

```bash

```

## 원인 추정

초기에는 다음 원인을 의심

- Token 오류
- 권한 부족
- OpenClaw 실행 안됨
- Discord 연결 실패

하지만 모두 아님

---

# 4. 실제 원인 ①

## Message Content Intent 비활성화

Discord Developer Portal

```text
Bot
→ Privileged Gateway Intents
→ Message Content Intent
```

OFF 상태 발견

---

## 이유

Discord는 기본적으로 Bot이 일반 채팅 내용을 읽지 못하도록 제한

Intent가 꺼져 있으면:

```text
사용자 메시지
↓
Discord
↓
차단
↓
OpenClaw 도달 불가
```

---

## 조치

Message Content Intent 활성화

---

## 결과

여전히 응답 없음

---


이 항목 때문에 Discord Listener가 실행되지 않은 것으로 의심

실제 원인은 아니었음

# 6. 실제 원인 ②

## DM 테스트

Discord DM으로 Bot에게 메시지 전송

```text
Hello
```

응답:

```text
OpenClaw: access not configured.
Pairing code: DVK8N92V
```

---

## 의미

Discord 연결은 성공했지만

사용자 승인(Pairing)이 완료되지 않은 상태

---

## 해결

Ubuntu VM에서:

```bash
openclaw pairing approve discord DVK8N92V
```

실행

결과:

```text
Approved discord sender ...
Command owner configured ...
```

# 이번 실습에서 배운 점

## Discord 연동은 4단계

```text
1. Bot 생성
2. 서버 초대
3. Message Content Intent 활성화
4. Pairing 승인
```

---

## Pairing이 핵심

많은 시간을 Discord 설정 문제로 생각했지만

실제 마지막 원인은

```text
OpenClaw Pairing 미승인
```

이었다.
