## Codex 분석

응답

```text
HTML을 raw 문자열로 반환하여 JavaScript가 정상 파싱되고
addTodo()가 정의됩니다.
```

---

## Codex 수정 내용

변경 전

```python
return """
```

변경 후

```python
return r"""
```

---

## 수정 이유

기존

```python
return """
<html>
...
<script>
...
</script>
"""
```

Python 문자열 처리 과정에서 HTML/JavaScript 일부가 의도치 않게 해석될 수 있음

---

Raw String 사용

```python
return r"""
...
"""
```

↓

HTML 원본 그대로 전달

↓

JavaScript 정상 파싱

↓

addTodo() 정상 인식

## Scale Up

```text
CPU 2
RAM 8GB

↓

CPU 4
RAM 16GB
```

---

## Scale Out

서버 수 증가

```text
1대
↓

26대
```

화면에서 최대 26개 인스턴스 확인

---

# 13. 스케일 아웃 방식

세 가지 모드 확인

## 수동

```text
항상 지정한 수 유지
```

예)

```text
26대 고정
```

---

## 자동

```text
트래픽에 따라 자동 증설
트래픽 감소 시 자동 축소
```

예)

```text
사용자 증가
↓
1대 → 2대 → 4대

사용자 감소
↓
4대 → 1대
```

---

## 규칙 기반

예)

```text
CPU > 70%
↓
인스턴스 +1

CPU < 30%
↓
인스턴스 -1
```
