# Agentic AI 과정 - 5교시 정리
## Azure AI Search + Vector Search + Embedding

### 1. Azure Storage 복습

Azure Storage Account 내부에는 여러 종류의 저장소가 존재한다.

#### Blob Storage
- 객체(Object) 저장소
- PDF, 이미지, 영상 등의 파일 저장
- Container 단위로 관리
- Container 하나를 디스크처럼 생각해도 무방

#### File Share
- SMB 3.0 지원
- NAS(Network Attached Storage)와 유사
- 네트워크 드라이브처럼 사용 가능

#### Azure Data Box

대용량 데이터(수십~수백 TB)를 물리 장비로 Azure에 이전하는 방식이다.

```mermaid
flowchart LR
    A["회사 데이터센터"] -->|데이터 복사| B["Azure Data Box"]
    B -->|택배 발송| C["Microsoft 데이터센터"]
    C --> D["Azure Storage"]
```

### 3. Azure AI Search 데이터 처리 흐름

```text
원본 데이터(Data Source)
        ↓
     Indexer
        ↓
      Index
        ↓
     검색(Query)
```

Data Source:
- Blob Storage
- Database
- 기타 정형/비정형 데이터

Indexer:
- 원본 데이터 확인
- 문서 변경사항 감지
- Index 자동 갱신

Index:
- 검색을 위한 저장 구조

```mermaid
flowchart LR
    DS["Data Source<br/>(PDF, DB, Files)"] --> IDXER["Indexer"]
    IDXER --> IDX["Index"]
    USER["User"] --> IDX
```

### 6. Chunk 와 Chunking

#### Chunk

문서를 잘라서 만든 조각

예)

```text
Chunk #1
Chunk #2
Chunk #3
```

#### Chunking

문서를 여러 Chunk로 분할하는 과정

```mermaid
flowchart LR
    PDF["PDF"] --> C1["Chunk 1"]
    PDF --> C2["Chunk 2"]
    PDF --> C3["Chunk 3"]
    C1 --> EMB["Embedding Model"]
    C2 --> EMB
    C3 --> EMB
    EMB --> V1["Vector"]
    EMB --> V2["Vector"]
    EMB --> V3["Vector"]
```

---

### 9. Embedding의 역할



---

사용자 질문

```text

```

↓

↓

유사도 계산

↓

관련 문서 검색

#### 문서 등록 과정

```mermaid
flowchart LR
    PDF["PDF 문서"] --> CHUNK["Chunking<br/>1024 Tokens"]
    CHUNK --> EMB["text-embedding-3-small"]
    EMB --> VECTOR["1536차원 Vector"]
```

```mermaid
flowchart LR

    VECTOR --> RESULT["관련 Chunk"]
```

## 오늘의 핵심

### 최종 Vector Search 구조

```mermaid
flowchart LR
    PDF["PDF"] --> CHUNK["Chunking<br/>1024 Tokens"]
    CHUNK --> EMB["text-embedding-3-small<br/>1536 Dimensions"]
    EMB --> VECTOR["Vector Store"]
```

### Azure AI Search 전체 RAG 파이프라인

```mermaid
flowchart LR
    PDF["PDF 문서"] --> CHUNK["Chunking"]
    CHUNK --> EMB["text-embedding-3-small"]
    EMB --> VECTOR["Vector Index"]
    QUERY["사용자 질문"] --> EMB
```

- PDF와 질문 모두 동일한 Embedding 모델을 사용한다.
- Vector Search를 수행하는 RAG의 핵심 구조다.

```text
PDF
↓
Chunking
↓
Embedding (text-embedding-3-small)
↓
1536차원 Vector
↓
Azure AI Search Index
↓
Vector Search
↓
RAG
```

오늘은 Azure AI Search와 Embedding Model을 이용하여
실제 RAG 시스템이 동작하는 핵심 원리를 학습하였다.
