# 1. Mock Function Calling → Real Function Calling

지금까지 만든 Tool들은 대부분 Mock 형태였다.

예시:

```python
def get_weather(location):
    conditions = ["맑음", "흐림", "비", "눈"]
```

```python
def get_exchange_money(...):
    exchange_rates = {...}
```

실제 외부 시스템을 조회하지 않고 프로그램 내부의 데이터만 사용하였다.

---

Real Function Calling은 실제 시스템과 연결한다.

예시

- 실제 날씨 API
- 실제 환율 API
- 실제 데이터베이스
- 실제 파일
- 실제 웹 서비스

구조

사용자
↓
LLM
↓
Tool
↓
실제 외부 시스템

# 3. SLA(Service Level Agreement)

SLA는 서비스 제공자가 보장하는 가용성 수준이다.

예시

- 99.9%
- 99.95%
- 99.99%

---

중요한 점

SLA는

"장애가 발생하지 않는다"

가 아니라

"약속한 수준보다 장애가 많으면 보상한다"

라는 계약이다.

---

예시

99.9% SLA

↓

허용 장애 시간을 초과

↓

서비스 크레딧 지급

---

# 4. 가용성(Availability)

가용성이란

장애가 발생해도 서비스를 계속 제공할 수 있는 능력이다.

---

단일 VM

사용자
↓
VM1

VM1 장애

↓

서비스 중단

---

다중 VM

사용자
↓
Load Balancer
↓
VM1
VM2

VM1 장애

↓

VM2가 계속 서비스

---

핵심

서버가 죽지 않는 것

×

서버가 죽어도 서비스가 계속되는 것

○
